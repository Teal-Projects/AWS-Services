h1. Description 

Amazon Kinesis Streams allow you to ingest real-time data from various sources, such as your application logs or website clickstreams, and format it for your databases and data warehouses. 

h1. Main Features 
 
h5. example : 
* Purchases from online stores (like amazon.com) 
* Stock Prices 
* Game data (as the gamer plays) 
* Social network data 
* Geospatial data (like uber.com) 
* iOT sensor data 

h5. Kinesis can : 
* Send streaming data to 
* Load ans analyze streaming data 
* providing the ability for you to build your own custom applications for you business needs. 

h5. Types : 
* Kinesis Streams 
* Consist of shards. 
* Data have persistance. 
* The data capacity of your stream is a function of the number of shards that you specify for the stream. 
* Kinesis Firehose 
* Data is analyse on the fly by a lambda function and then save by S3, or elasticsearch cluster. 
* No Data parsistency. 
* Kinesis Analytics 
* Can Analyse data on the fly inside Kinesis Stream and Firehose
