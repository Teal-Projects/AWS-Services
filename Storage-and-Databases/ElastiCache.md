h1. Description 

Caching is a technique with the help of which very frequently required/accessed files/data (images, web pages) is stored in a temporary memory location on the server. Caching is used in most of the applications and one of the best use-case for this is applications that are read-intensive.

h1. Main Features 

h5. General :
* Increase database and web application performance
* ElastiCache Node :
** Node is the basic bloc. They are RAMs attached to one another in the network. They can be uniquely identified with the help of a DNS name and port.
* ElastiCache cluster :
** A collection of nodes. The working of cluster depend wether it is Memcached or Redis.
* Redis is Multi-AZ
* You can do a backup and restores of Redis
* If you want something simple and easy to get started use memcached
